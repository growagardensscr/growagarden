## 🛠️ Troubleshooting Common Issues

**❌ GUI doesn't appear after executing the script**
The overlay failed to initialize, usually because injection happened before the game finished loading.
**Solution:** Detach the executor, wait until you see your character on the map, then re-attach and execute again.

---

**❌ Pet Spawner runs but no pets appear**
The spawn packet is being sent but the server isn't registering it — usually a timing issue.
**Solution:** Increase `SpawnDelay` to `1.0` in `...config.lua` and re-execute. If the issue persists, try a different server region.

---

**❌ `...config.lua` not saving between sessions**
File write permissions are blocked by Windows User Account Control on protected paths.
**Solution:** Move the script folder from `C:/Program Files/...` to `C:/Desktop/GrowAGarden/` and run the executor as Administrator.

---

**❌ Anti-AFK not working — getting kicked after 20 minutes**
Some servers use custom idle detection that ignores standard anti-AFK pings.
**Solution:** Enable `ClickFarm` alongside Anti-AFK — the simulated click activity satisfies even custom idle detectors.

---

**❌ Script crashes on iOS after ~15 minutes**
iOS memory management is aggressive and may terminate background executor processes.
**Solution:** Enable **Low Power Mode OFF** while running the script, and close all background apps to free up RAM before launching.

---

**❌ Seed Spawner fills inventory with duplicates only**
The seed filter is matching your existing inventory and prioritizing owned seeds.
**Solution:** Set `AutoInventory = true` in the config — this enables the de-duplication logic that filters already-owned seed types from the spawner queue.
